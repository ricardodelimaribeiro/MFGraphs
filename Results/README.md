# Results

This directory holds generated benchmark outputs, validation reports, and
paper-support artifacts. Treat generated outputs as reproducible run artifacts:
do not hand-edit result data unless correcting a known encoding/export issue.

## Active Outputs

### `reduce_system_latest.csv`

- Generated by: `Scripts/BenchmarkReduceSystem.wls`
- Purpose: latest active benchmark summary for the current scenario-kernel
  structural solver path (`scenario -> makeSystem -> reduceSystem`)
- Timestamped copies: `reduce_system_<YYYYMMDD-HHMMSS>.csv`
- Columns: `Case`, `BuildMs`, `WarmupMs`, `RepMs`, `Status`, `Kind`, `Valid`

### `reduce_system_solutions_latest.wl`

- Generated by: `Scripts/BenchmarkReduceSystem.wls`
- Purpose: latest stored `reduceSystem` solutions and validation status
- Timestamped copies: `reduce_system_solutions_<YYYYMMDD-HHMMSS>.wl`
- Commit policy: selected solution snapshots are tracked when they document a
  benchmark/history entry

## Archived Or Legacy Outputs

Most other files in this directory come from legacy solver-era scripts, paper
validation runs, or one-off profiling work. They remain useful for historical
comparison, but they are not the active benchmark workflow while legacy symbols
such as `DataToEquations`, `CriticalCongestionSolver`, and `GetExampleData`
remain outside the default package load path.

Common legacy outputs:

| Files | Source | Notes |
|---|---|---|
| `benchmark_latest.{csv,json}`, `benchmark_<timestamp>.{csv,json}` | `Scripts/archive/BenchmarkSuite.wls` | Legacy tiered benchmark data |
| `bottleneck_report.md`, `bottleneck_data.json` | `Scripts/archive/BottleneckReport.wls` | Legacy bottleneck profiling |
| `compare_dnf.md`, `compare_dnf.json` | `Scripts/archive/CompareDNF.wls` | DNF reduction comparisons |
| `jamarat_phase_*` | `Scripts/archive/JamaratFeasibilityPhaseDiagram.wls` | Phase-diagram data |
| `preprocessing_profile_report.md` | `Scripts/archive/ProfilePreprocessing.wls` | Legacy preprocessing profile |
| `reduction_strategies_profile.md` | archived profiling scripts | Disjunct-reduction strategy comparison |
| `validation_*.md`, `paper_*`, `master_benchmark_summary_*` | validation/research scripts | Paper and research support artifacts |

See `Scripts/LEGACY_SOLVER_SCRIPTS.md` and
`Scripts/LEGACY_GETEXAMPLEDATA.md` before rerunning archived workflows.

## Tracking Policy

The repository ignores generated `Results/*.csv`, `Results/*.json`, and
`Results/*.png` files by default. Tracked files in this directory are intentional
reference artifacts, report snapshots, or selected Wolfram Language solution
snapshots.

Current tracked files can be listed with:

```bash
git ls-files Results
```

Historical performance logs live under `docs/history/`, not in `Results/`.

## Audit Notes

- `latest` files should match the most recent timestamped file from the same
  generator.
- JSON outputs should parse with `python3 -m json.tool`.
- Avoid Unicode placeholders in generated CSV/WL data; use ASCII `-` for missing
  values so exports stay stable across Wolfram and shell encodings.
